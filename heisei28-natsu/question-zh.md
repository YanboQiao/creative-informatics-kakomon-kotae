# 编程（中文）

请为以下问题编写程序。编写程序时，不得使用直接实现题目要求的内置库函数。例如，第 (1) 题的程序不能使用 Ruby 的 `to_i`。

(1) 编写程序读取一个四进制数（以 4 为基的位值表示法），并输出其十进制值。例如输入 123 输出 27。

(2) 设符号 a, b, c, …, h 分别表示数字 0, 1, 2, …, 7。编写程序读取一个使用 a, b, c, …, h 表示的八进制数，并输出对应的十进制值。例如输入 bcd 输出 83。

(3) 在答题纸上将十进制数 2015 写成罗马数字。

罗马数字使用七个符号：I, V, X, L, C, D, M，其值分别为 1, 5, 10, 50, 100, 500, 1000。形成规则如下：

数字通过组合符号并求和得到。例如 II 表示两个 1 即 2，XIII 表示 10 加三个 1 即 13。该系统没有 0，例如 207 表示为 CCVII（两个百、一个五、两个一）。1066 表示为 MLXVI（一千、五十、十、五、一）。

符号从左到右按大小排列，最大者在前。但为避免连续四次重复（如 IIII 或 XXXX），使用以下减法规则：

- I 可放在 V 和 X 前，构成 4（IV）和 9（IX）  
- X 可放在 L 和 C 前，构成 40（XL）和 90（XC）  
- C 可放在 D 和 M 前，构成 400（CD）和 900（CM）

例如：1904 = 1000 + 900 + 4，分别写作 M + CM + IV，因此为 MCMIV。

（参考： http://en.wikipedia.org/wiki/Roman_numerals ）

要求符号数量最少。例如 IV 仅由两个符号组成。

(4) 编写程序读取一个罗马数字并输出其十进制值。假设数字大于 0 且小于 4000。

(5) 编写程序读取一个十进制数并输出其罗马数字形式。数字大于 0 且小于 4000。

(6) 扩展罗马数字的减法规则如下：

一个符号可放在两个更大符号 $\alpha$ 与 $\beta$ 之间，其中 $\alpha > \beta$，或 $\alpha$ 为空。例如：

- CIL 表示 $100 - 1 + 50 = 149$  
- IL 表示 $-1 + 50 = 49$  

编写程序读取一个十进制数，并输出其扩展罗马数字表示。数字大于 0 且小于 4000。要求符号数量最少。

(7) 编写程序读取一个以英文书写的数字，并输出其十进制形式。数字是小于 100000 的正整数。例如：

fifty four thousand three hundred twelve

输出 54312。程序只需接受每个数字至少一种英文表达。例如对于 1200，可只接受 one thousand two hundred 或 twelve hundred。
