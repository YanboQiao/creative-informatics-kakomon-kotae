# 编程

大型程序中经常包含重复代码，称为代码克隆。我们将编写一个简单程序，用来检测给定文件中的重复代码。

(1) 编写一个程序，读取 U 盘中的 `program.txt` 文件，并统计其中分号 `;` 的个数。`program.txt` 是一个文本文件。

(2) 编写一个程序，读取 `program.txt` 并打印所有包含 `main` 的行以及它们的行号。

(3) 编写一个程序，读取 `program.txt` 并打印文件中**连续重复的行**。重复行只打印一次。例如，若文件内容如下：

a = 1  
a = a + 1  
a = a + 1  
b = a  
a = 1  

程序应输出：

a = a + 1  

注意：它不会打印 `a = 1`，因为这两行并非相邻。

(4) 编写一个程序，读取 `program.txt` 并打印文件中所有重复行。即使重复的第二次出现不与第一次相邻，也要识别出来。以上述例子为例，程序应打印：

a = 1  
a = a + 1  

每条重复行只输出一次，且按第一次出现的顺序打印。此外，程序必须打印重复行的数量。以上例中数量为 $2$。注意，同一行可能出现两次以上。

(5) 编写一个程序，读取 `program.txt` 并打印所有**相似的两行**，同时打印找到的“相似行对”的数量。这两行不要求相邻。忽略长度短于 $20$ 个字符的行。

两行被定义为相似，当它们不完全相同且对应字符不同的位置数少于 $5$ 个。若行长度不同，则在较短的行末加空格，使两行长度一致。例如（为了简单起见示例行少于 $20$ 字符）：

a = 1  
a = a + 1  

对应字符不相同的位置数为 $3$：第 $5$、$7$、$9$ 个字符不同。前四个字符以及 `+` 两侧的空格相同。

(6) 修改 (5) 中的程序，**重新定义相似行**。新的定义为：将一行变为另一行所需的最少编辑操作步骤数小于 $4$ 时，两行相似。完全相同的行不算相似。编辑操作包括：替换一个字符、删除一个字符、插入一个字符。

(7) 编写一个程序，读取 `program.txt`，当文件中出现由多于三行组成的**连续重复行组**时，打印这些行。行必须完全相同才算重复。每个重复组仅打印一次。
